XML & XSL

The following samples render various XML documents in the browser using XSL and CSS. The source code for each can be viewed through my GitHub democode repository.

  • Sonnet 29
    This is one of my favorite Shakespeare sonnets. And what better demonstration of a document type definition (DTD) than a sonnet? A sonnet isn't just a poem. A sonnet has a well defined structure with formal names for its parts: quatraine, couplet, rhyme scheme etc. I express these formal elements in my DTD.
  • A short list of films
    A film description offers an interesting example for a formal document definition. Not only can a film be described with rich metadata, but that data is often context-specific. Consider dates. A date associated with a film could describe when the film was produced or the setting date. Consider awards. Awards can be give to the film - ex. National Film Registry - or to individual cast and crew members - ex. best cinematography. This document's definition expresses such detail using optional and implied elements. The XLT transformation handles proper labeling and punctuation of this optional content.
  • A library catalog record
    This document is in MaRC XML format. MaRC is the standard metadata format for library catalog records and was developed by the Library of Congress. The MaRC metadata schema is very granular and identifies fields with alpha-numeric codes. This document is styled so that the data column has an approximately 50 character measure for readability.
  • Broome Community College Top 100 Books 2008
    The XML of this sample is a report from Aleph - a popular library catalog system. The results contain 355 books with high usage, but the report did not have options for limiting the results to 100 or sorting. Librarians often open these XML reports in MS Excel and tweak them for presentation. But, in this case I created an XSL transformation to perform the sort and limit the list to 100 items. Cover images come from the Google Books API based on ISBN's found in the report data.
  • Upstate Medical University METS record
    Metadata Encode and Transmission Standard (METS) is a standard maintained by the Library of Congress and is commonly used in digital libraries and archives. METS documents are encoded with a namespace and often include other metadata schemata such as MODS and Dublin Core. This sample simply lists each datum with its element name and any relevant modifier attributes.
  • Mildred Davenport Dance Programs EAD Finding Aid
    Encoded Archival Description (EAD) is another standard maintained by the Library of Congress. It is used to create inventories of archival materials. The EAD schema includes 146 elements many of which are optional and can be nested. The flexibility of the schema presents a challenge when displaying documents, as they can be long and complex. For this demo I created a rendering that works for any EAD document. XSL is used to transform all elements to nested HTML divs while jQuery .toggle() is used to animate the display as a collapsable tree. In practice it is probably best to render individual EAD documents with custom stylesheets, but this example demonstrates a quick way to display any EAD document.
